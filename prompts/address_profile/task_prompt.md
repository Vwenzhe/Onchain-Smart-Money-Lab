# 地址画像 Task Prompt

本文档提供地址画像能力的 `task prompt` 模板，适合：

- 单条 API 调用
- 批处理任务
- 后续脚本内通过模板变量渲染

建议与 [system_prompt.md](file:///D:/aiwork/DprojectsAAA-project-1/prompts/address_profile/system_prompt.md) 搭配使用。

## 1. 通用 Task Prompt 模板

将下面模板作为 `user` 消息发送给模型：

```text
请基于以下地址结构化特征，生成该地址的行为画像。

任务要求：
1. 只能依据输入字段进行判断，不得补充输入中不存在的背景信息。
2. 输出必须克制、可解释、基于证据。
3. 不允许输出无证据断言。
4. 不允许输出交易建议、价格预测、身份判断或绝对因果结论。
5. 若证据不足，请使用 `待定观察型`，并在 `risk_note` 与 `summary` 中明确说明不确定性。
6. 允许客观描述持仓变化、仓位变化、持续建仓或未明显减仓，但不得形成任何买卖或仓位操作建议。

请按指定 JSON 格式输出结果。

输入数据：
{{ADDRESS_FEATURE_JSON}}
```

## 3. 批处理场景模板

适用于离线批量生成画像结果。建议一条地址一轮调用，避免多地址混在一个输出中增加解析复杂度。

```text
你将处理一条地址画像任务。请严格依据输入 JSON 为单个地址生成画像结果。

批处理上下文：
- dataset: address_feature_snapshot
- task_type: address_profile_generation
- output_fields: profile_label, risk_note, summary
- strict_json_only: true

任务要求：
1. 只能使用当前输入记录中的字段。
2. 不得引用其他地址、历史轮次或外部知识。
3. 若证据不足或字段缺失严重，输出 `待定观察型`。
4. 输出必须是单个 JSON 对象。

输入记录：
{{ADDRESS_FEATURE_JSON}}
```

## 4. 输出格式约束

模型输出必须满足以下要求：

```json
{
  "profile_label": "趋势跟随型",
  "risk_note": "当前浮盈较高，后续若交易节奏转弱，需关注是否出现集中兑现。",
  "summary": "该地址在观察窗口内呈现出较明确的顺势参与特征，净流入、持仓价值和当前浮盈均处于较高水平，更接近趋势跟随型账户。当前判断基于单次结构化快照，仍需继续观察后续行为一致性。"
}
```

附加约束：

- `profile_label` 必须是以下之一：
  - `早期埋伏型`
  - `趋势跟随型`
  - `深度持有型`
  - `高浮盈型`
  - `高活跃轮动型`
  - `待定观察型`
- `risk_note` 必须是字符串，建议 20 到 60 字。
- `summary` 必须是字符串，建议 40 到 120 字。
- 不得输出 `null`、数组、嵌套对象或额外字段。
- 不得输出 Markdown、代码块、编号、解释文本。

## 5. 错误输出兜底策略

建议在调用侧按以下顺序兜底：

### 5.1 解析失败

若模型输出不是合法 JSON：

1. 先尝试提取首个 JSON 对象并重新解析。
2. 若仍失败，使用固定兜底结果：

```json
{
  "profile_label": "待定观察型",
  "risk_note": "模型输出格式异常，当前结果已降级为待定观察，需结合原始特征重新生成。",
  "summary": "当前地址画像未能稳定生成结构化结果，因此暂归为待定观察型。建议在保留原始输入的前提下重新调用模型，并检查输出格式约束是否生效。"
}
```

### 5.2 字段缺失

若缺少任一字段，或字段类型不是字符串：

1. 优先触发一次低温重试。
2. 重试仍失败时，使用固定兜底结果：

```json
{
  "profile_label": "待定观察型",
  "risk_note": "当前画像字段不完整，暂无法形成稳定判断，需继续观察后续行为特征。",
  "summary": "由于本次模型输出缺少必要字段，系统已将该地址降级为待定观察型，以避免产生不受控的过度解释。"
}
```

### 5.3 标签越界

若 `profile_label` 不在允许集合内：

1. 若语义接近允许标签，可在调用侧映射到最近标签。
2. 若无法安全映射，直接降级为 `待定观察型`。

### 5.4 内容越界

若输出中出现以下任一问题，应直接丢弃本轮结果并降级：

- 交易建议
- 价格预测
- 身份断言
- 绝对因果判断
- 明显脱离输入字段的外推

建议固定兜底结果：

```json
{
  "profile_label": "待定观察型",
  "risk_note": "当前输出超出证据边界，已按保守策略降级处理，需继续观察并重新生成。",
  "summary": "本次画像结果包含超出输入证据范围的推断，为保证可解释性与稳定性，系统暂将该地址归为待定观察型。"
}
```

```json
  "summary": "该地址在观察窗口内保持了一定活跃度，当前净流入、持仓价值和浮盈比例均处于较高水平，更接近顺势参与并跟随行情的趋势跟随型账户。当前判断仅基于结构化快照，仍需结合后续行为连续性继续观察。"
}
```

## 7. 推荐调用参数

为提高稳定性，建议：

- `temperature`: `0.1` 到 `0.3`
- `top_p`: 如模型支持，建议 `<= 0.9`
- 优先选择指令遵循较强的小中型模型
- 单轮只处理一个地址，避免输出串行污染
