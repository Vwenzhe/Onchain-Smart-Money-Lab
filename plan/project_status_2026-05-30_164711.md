## 4. 当前主链路

当前 Python 主链路如下：

`config/tokens.json -> scripts/render_sql.py -> scripts/run_dune_queries.py -> scripts/build_feature_layer.py`

总入口脚本：

它负责按顺序执行：

1. 渲染 SQL
2. 执行 Dune 参数化查询
3. 生成特征层结果

## 5. 各 Python 脚本职责说明

### 5.1 `scripts/run_pipeline.py`

用途：

- 一键执行完整流程

适用场景：

- 需要从配置开始，一次性跑到特征层结果时使用

### 5.2 `scripts/render_sql.py`

用途：

- 读取 `config/tokens.json`
- 将模板 SQL 渲染到 `sql/rendered/<TOKEN>/`

适用场景：

- 只想确认 SQL 模板是否按 token 正确展开

### 5.3 `scripts/run_dune_queries.py`

用途：

- 根据 token 配置，执行 Dune 参数化 Query
- 将结果写入 `data/raw/dune/<TOKEN>/`

适用场景：

- 只想刷新 Dune 原始结果

依赖：

- 根目录 `.env`
- `DUNE_API_KEY`

### 5.4 `scripts/build_feature_layer.py`

用途：

- 读取 `data/raw/dune/` 下的原始 JSON
- 生成稳定结构的 processed 结果

适用场景：

- 原始结果已有，只想重新生成特征层

注意：

- 该脚本当前是按 `data/raw/dune/*` 目录扫描
- 它不会按 `tokens.json` 再做一次 token 过滤

### 5.5 `scripts/test_dune_latest_result.py`

用途：

- 测试某个 `query_id` 的最新结果能否拉取成功

适用场景：

- 用于排查 Dune API 或单个 Query 是否可用

说明：

- 这是调试脚本，不属于正式主流程

## 7. 如果只想跑 ETH，如何操作

当前项目没有单独的命令行参数来指定 token，因此只跑 `ETH` 的方式是通过配置与数据目录共同控制。

### 7.1 修改配置

编辑：

- `config/tokens.json`

操作原则：

- 仅保留 `ETH` 的 `is_enabled` 为 `true`
- 将 `FET` 和 `PEPE` 的 `is_enabled` 改为 `false`

### 7.2 清理旧原始结果

因为 `build_feature_layer.py` 会扫描已有的原始结果目录，所以如果此前已经跑过 `FET` 或 `PEPE`，需要删除或移走：

- `data/raw/dune/FET/`
- `data/raw/dune/PEPE/`

### 7.3 如需只保留 ETH 的 processed 结果

还应删除旧的非 ETH 处理结果文件，避免混淆。

### 7.4 重新执行

执行：

```bash
python scripts/run_pipeline.py
```
